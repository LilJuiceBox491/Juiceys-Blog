What Is Markdown, It's Usefulness, And Why You Should Use It

Markdown. A lot of you have at least heard of it. But what is it? Why is it useful? Why should you use it?

Markdown is 'a plain text formatting syntax aimed at making writing for the internet easier'. In other words, it's fancy symbols you squish in the middle of your text to make it fancy.

That doesn't make it useless though. Many long text inputs for applications of any sort on the internet support Markdown, so take advantage of it! Learn markdown (it is really simple), and impress your friends/family/colleagues with your fancy text skills.

Markdown is really simple. Just learn a few basic symbols that you wrap your text in, and voila! You are a markdown genius!

One more commonly used place of Markdown is README.md's. Many open-source repos have a README file in markdown to teach the reader/viewer what their project is and how to use it.

All in all, Markdown is useful. It doesn't take too much effort to learn, and it'll pay off when you need it.

My Thoughts On Scratch to JavaScript

News has been floating around lately about Scratch adding a JavaScript feature. A button that you click that converts your block code to JavaScript. This is because people say Scratch isn't great for learning what's known as 'real programming'.

I think this is a pretty cool idea. I mean, imagine if you could take your beloved Scratch code and turn it into useful JavaScript! However, my opinion is sort of biased, if you will.

I am a JavaScript programmer, so for me this feature would be vrey useful, but for many others, is it that useful? No. Many Scratcher's wouldn't even use this feature. So is it worth the effort to create for only a small amount of people using it?

My final decision is YES. The people who would use this a lot (like me), would love it! It would change the way we used Scratch entirely. Something like this in the Scratch 4.0 update can really help bring Scratch's old hype back up and prepare young coders for the future of programming.
